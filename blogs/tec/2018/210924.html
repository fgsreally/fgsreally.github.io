# 二进制结构

# 标准模式(困难)

先从这一段看起

var buffer = new ArrayBuffer(12);

var x1 = new Int32Array(buffer);
x1[0] = 1;
var x2 = new Uint8Array(buffer);
x2[0]  = 2;

x1[0] // 2`

相当于arraybuffer得到一块二进制数据，然后再在这个长度自由的部分上面利用视图去写入固定长度的东西

这玩意儿我只见过两种用法

（虽然按理来讲应该很有用，因为这个类型是跟c一致的，但如果需要和c交互，更多还是用专用的库去做类型转换，如ref）

  • 视觉方面的，webgl每写一个东西都是arraybuffer进出缓冲区 其实我感觉3.js好处就是把这个底层部分屏蔽了。。太糟心了 还有canvas那个元素操作也是

  • 请求方面的，也就是fetch，ajax，websocket那些，我写过一个基于protobuf的前后端微服务一整套的东西，就是要在响应体里面搞出来个arraybuffer。

完全手动操作这个挺不靠谱的，还是要有某个约定来进行读取， 不想操作的话就最好直接响应体为blob

# 简易模式(不需要知道结构究竟是几位的)

像blob这个对象，就有点像基于约定的arraybuffer，变成一个js对象，用起来就舒服多了，直接往里面导就行了，切片什么的直接push进去就拼凑起来了，读size属性就看到长度， 配合URL.createObjectURL(blob)直接用就行

注意要URL.revokeObjectURL()释放掉

URL.createObjectURL()与FileReader.readAsDataURL()功能差不多 但后者返回的是超长base64字符串，与文件大小相关
